3. Setup instructions/rules bằng Kiro steering

Kiro dùng steering cho instruction/rule. Nên tách global và project để tránh pollute mọi workspace.

3.1 Global steering

Dùng cho các rule luôn áp dụng giống global CLAUDE.md:

C:\Users\Thanh\.kiro\steering\global.md

Nội dung nên port từ C:\Users\Thanh\.claude\CLAUDE.md, nhưng rút gọn cho Kiro:

# Global Agent Instructions

- Communicate in Vietnamese by default.
- Be concise, practical, and explicit about uncertainty.
- Prefer simple, surgical changes.
- Do not overwrite user files or run destructive commands without confirmation.
- Do not commit, create branches, or open PRs unless explicitly requested.
- Use only Claude/Kiro-native agents/tools; do not route to Codex/Gemini.
- Keep specs/plans/files inside the current workspace unless explicitly approved.
- Do not hardcode secrets in config files.
- For non-trivial coding work: plan briefly, verify with commands or runtime checks, then report evidence.

5. Setup memory

Claude Code hiện dùng Memorix hook + MCP. Kiro không có cùng hook runtime đã xác nhận, nên cách gần nhất là dùng memorix như MCP server.

5.1 Cấu hình MCP

{
  "mcpServers": {
    "memorix": {
      "command": "memorix",
      "args": ["serve"],
      "disabled": false,
      "autoApprove": []
    }
  }
}

5.2 Steering để buộc thói quen memory

# Memory Protocol

- At session start, check whether project memory is available.
- Before asking user for old context, search memory first when possible.
- When discovering important non-obvious project information, store a concise discovery memory if the memory MCP is available.
- Never store secrets or temporary task state as durable memory.

Giới hạn: Memorix trên máy này yêu cầu git-backed project để bind project scope. Workspace không có git hoặc bị safe.directory issue có thể làm memory fail.

kiro D:\Job\AI\setup-kiro

# Kiểm tra thư mục global Kiro
Get-ChildItem C:\Users\Thanh\.kiro -Force
Get-ChildItem C:\Users\Thanh\.kiro\steering -Force

PowerShell có thể cần escape JSON khác nhau theo shell. Nếu lỗi quote, ưu tiên tạo file mcp.json thủ công thay vì dùng --add-mcp.

9. Thứ tự triển khai đề xuất

  1. Không port secret. Đưa API key/token sang environment variables trước.
  2. Tạo C:\Users\Thanh\.kiro\steering\global.md từ rule ngắn gọn của CLAUDE.md.
  3. Tạo project .kiro\steering\project.md cho từng workspace cần dùng Kiro.
  4. Tạo C:\Users\Thanh\.kiro\settings\mcp.json với 1-2 MCP dễ test trước: memorix, gitnexus.
  5. Mở Kiro, kiểm tra MCP tools xuất hiện và chạy được.
  6. Thêm dần các MCP có dependency nặng: zai-mcp-server, tradingview, cloakbrowser.
  7. Sau khi ổn, cân nhắc project-level MCP override ở .kiro\settings\mcp.json cho từng repo.
  8. Chỉ bật autoApprove sau khi đã biết rõ tool nào an toàn.
